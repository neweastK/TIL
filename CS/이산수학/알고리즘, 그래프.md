## 알고리즘

> 문제를 해결하기 위한 절차를 기술한 것
>
> 누구나 정해진 절차대로 실행할 수 있어야함
>
> 누구나 동일한 입력값이 주어지면 이 절차대로 실행하면 동일한 출력값을 얻을 수 있어야함
>
> 일정한 시간안에 출력값을 구할 수 있어야함



#### 특징

- 순서대로 정의된 절차
  - 분명한 순서가 있어야함
  - 한 동작을 실행하면 다음에 실행할 동작이 무엇인지 분명해야함
- 명확성
  - 모든 동작은 명확하게 정의되어야함
  - 모든 동작은 실행 가능해야함
- 반드시 원하는 결과가 나와야함
- 일정 시간 안에 나와야함



#### 구조

1. 순차적 구조(sequential structure)
   - 각 단계별 순서대로 수행하는 것
   - 가장 기본적인 구조
2. 분기 구조(branch)
   - 특정 조건에 따라 수행하는 단계가 달라지는 것
3. 반복 구조(repetition)
   - 어떤 조건을 만족할 때, 절차를 반복하는 것
4. 점프 구조(jump)
   - 순차적으로 수행하다가 특정 절차로 이동하는 것



#### 기술 방법

1. 플로우 차트
   - 도표로써 알고리즘을 기술할 수 있음
   - but, 복잡한 절차를 표현할 경우 쉽게 이해하기 어려움
2. 프로그램 언어의 코드
   - 해당 언어의 문법을 알아야함
3. Pseudo Code
   - 프로그램 언어 코드의 전 단계로 프로그램 언어들이 사용하는 구조를 그대로 이용



#### 알고리즘의 검증

- 귀납법 활용
  - 몇가지 사례에 대하여 옳음을 보여야함
    - 일반적인 값
    - 특수한 입력값



## 그래프

> 왜 그래프가 중요한가?
>
> 	- 많은 문제들이 그래프로 모델화가 됨
> 	- 이러한 그래프로부터 알고리즘을 찾게됨



- 그래프란?
  - 정점의 집합과 정점들을 연결하는 선들의 집합
  
- 인접(adjacent)

  - 임의의 연결선 e = (u,v)에 대해서 정점 u와 v는 서로 인접했다고 함
  - 이때, e는 정점 u와 v에 접합(incident)한다고 함

- 루프(loop)

  - 연결선의 두 끝점이 같은 정점

- 다중 연결선

  - 두 정점의 연결선이 두 개 이상일 때

- 단순 그래프(simple graph)

  - 루프나 다중 연결선이 없는 그래프

- 연결 그래프

  - 모든 정점들이 연결되어 있는 그래프

- 비연결 그래프

  - 모든 정점들이 연결되어 있지 않고, 일부만 연결되어 있는 그래프

- 차수(degree)
  - 하나의 정점에 접합된 연결선의 수
  - 정점 u에 대한 차수는 deg(u)와 같이 표기하기도 함
  - 모든 정점의 차수의 합은 모든 연결선 수의 2배

- 길이(length)

  - 두 정점의 경로를 구성하는 연결선의 수

- 거리(distance)

  - 두 정점 간의 최단 경로의 길이

- 닫힌 경로(closed path)

  - 경로 {V1,V2,....,Vn}에서 V1=Vn인 경로

- 순환(cycle 또는 circuit)

  - 3개 이상의 연결선을 갖는 경로에서 어떤 연결선도 중복되지 않는 닫힌 경로

- 부분 그래프(subgraph)

  - 그래프가 있을 때, 정점이 특정 그래프의 부분집합이고, 연결선도 해당 그래프의 부분집합일 경우 특정 그래프의 부분 그래프라고 함

- 동형 그래프(isomorphic graph)

  - 임의의 두 그래프 G={V,E}와 F={V2,E2}에 대하여 다음의 조건을 만족하는 함수가 1:1 관계의 함수이면 두 그래프는 동형그래프이다
    - 함수 f:V→V2
    - (x,y) ∈ E ⇔ (f(x),f(y)) ∈ E2
    - 위 관계가 성립하는 함수 f를 동형(isomophic)라고 한다

  - 즉, G 그래프의 정점을 F 그래프의 정점과 매칭을 시켜주고, 그 때, 간선들이 동일하다면 동형이라는 것
    - 간선들이 동일하다 == 동일한 접점 두개를 연결하고 있다

- 완전 그래프(complete graph)

  - 그래프의 모든 정점 사이에 연결선이 존재하면 이는 완전 그래프
  - 완전 그래프는 Km으로 표시 (m은 정점의 수)

- 이분 그래프(bipartite graph)

  - 그래프의 정점이 교집합이 없는 두 부분집합 X와 Y로 갈라지고, 연결선이 (X의 원소, Y의 원소)로만 이루어지면 이분그래프

  - X의 모든 정점과 Y의 모든 정점 사이에 연결선이 존재하면 완전 이분 그래프 

  - Km,n으로 표시 (m은 X의 개수, n은 Y의 개수)

- 정규 그래프(regular graph)
  - 모든 정점의 차수가 같은 그래프
- 평면 그래프(planar graph)
  - 연결선들이 서로 교차하지 않고 평면상에 그릴 수 있는 그래프
  - ↔ 비평면 그래프
  - 면(face) : 연결선에 따라 구분된 영역
- 방향 그래프(directed graph, digraph)
  - 연결선에 방향이 있는 그래프



### 오일러 그래프(Eulerian path)

> 오일러 순환이 존재하는 그래프

- 오일러 경로 (Eulerian path)

  - 그래프의 모든 **연결선**을 한번만 방문하는 경로

- 오일러 순환 (Eulerian cycle 혹은 Circuit)

  - 시작점과 끝점이 동일한 오일러 경로

- 즉, 오일러 그래프란 **시작점과 끝점이 동일하면서 그래프의 모든 연결선을 한번만 방문하는 경로가 존재하는 그래프**

- 오일러 경로를 갖기 위한 필요 충분 조건

  ```
  2개 이상의 정점을 갖는 루프가 없는 연결 그래프에서 홀수 차수를 갖는 정점이 하나도 없거나 오직 두개만 존재해야한다. 오직 두개만 홀수 차수라면 그 두개가 시작점과 끝점이다.
  특히, 모든 정점이 짝수 차수를 가지면 오일러 순환이 존재하며, 이 그래프는 오일러 그래프이다.
  ```

  

### 해밀턴 그래프

- 해밀턴 경로(Hamiltonian path)
  - 그래프에서 모든 **정점**을 정확히 한번만 지나는 경로
  - 하나의 정점을 두번 지날 수 없다
- 해밀턴 순환(Hamiltonian cycle 혹은 circuit)
  - 시작점과 끝점이 같은 해밀턴 경로
- 특정 그래프에서 해밀턴 순환을 찾는 알고리즘
  - 오일러 순환을 찾는 알고리즘은 있으나 해밀턴 순환을 찾는 알고리즘은 없음
  - 따라서, 모든 경우의 수를 시도해봐야함
- 대표적인 문제로는 방문 판매원 문제가 있음





### 그래프 채색

> 인접하고 있는 정점들은 서로 다른 색을 갖도록하면서 그래프의 모든 정점에 색을 할당하는 문제
>
> - 색상수 : 그래프 채색에 필요한 최소한의 색의 수
> - 예시
>   - 완전 그래프 Kn의 색상수는 n
>   - 이분 그래프 Km,n의 색상 수는 2



#### 그래프 채색 알고리즘

- 그리디 알고리즘을 활용
  - 그리디 알고리즘
    - 결정을 할 때마다 최종 결과에 상관없이 그 순간에서 최선의 선택을 한다
    - 그 순간의 선택은 그 순간에서 최적의 선택
    - but, 최종의 결과가 최적이라는 보장은 없음!
- 현재 노드의 인접한 노드들을 모두 탐색한 후 색칠 가능한 색깔 중 번호가 가장 낮은 색깔을 선택한다
  - 그 후, 다음 노드로 이동해서 위 과정을 반복한다
  - but, 항상 맞는 결과를 내주지는 않음
  - 노드 탐색 순서에 따라 다른 결과가 나옴
